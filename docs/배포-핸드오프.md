# 배포 도움 요청 (이어서 진행)

안녕! 나는 지금 내 프로젝트를 AWS에 **처음으로 배포**하는 중이야.
배포가 완전 처음이라 어려워. 그래서 **한 번에 한 단계씩, 천천히, 초보자 눈높이로** 설명해줘.
명령어를 줄 때는 **"이건 내 컴퓨터에서" / "이건 EC2 서버 안에서"** 인지 꼭 표시해줘.
한 번에 여러 단계를 쏟아내지 말고, 내가 한 단계 끝내고 "됐어"라고 하면 다음으로 넘어가 줘.

아래는 내 프로젝트 정보랑, 지금까지 진행한 상황이야.

---

## 1. 프로젝트가 어떻게 생겼나 (4개 조각)

```
   [브라우저]
       │
       ▼
 ┌──────────────┐
 │ ① 프론트엔드  │  React 19 + Vite (TypeScript) → 빌드하면 dist/ 정적 파일
 └──────┬───────┘
        │ /api 호출
        ▼
 ┌──────────────┐
 │ ② 백엔드      │  Spring Boot 4.0 / Java 21 / Gradle → 실행 가능한 fat jar
 └──┬────────┬──┘
    ▼        ▼
 ┌──────┐ ┌──────┐
 │③ DB  │ │④ S3  │  ③ PostgreSQL 16 (docker-compose로 컨테이너 실행)
 │      │ │      │  ④ 이미지 저장은 AWS S3 (AWS SDK v2)
 └──────┘ └──────┘
```

### 기술 스택 상세
- **백엔드**: Spring Boot 4.0, Java 21, Gradle, JPA + QueryDSL, Flyway(부팅 시 자동 마이그레이션, `ddl-auto=validate`)
- **인증**: JWT (access는 메모리, refresh는 쿠키) + OAuth2 (Google, Kakao)
- **프론트**: React 19, Vite 8, TailwindCSS 4, react-query, zustand
- **DB**: PostgreSQL 16 (현재 `docker-compose.yml`에는 postgres 컨테이너만 정의됨)
- **이미지 스토리지**: AWS S3 (리전 `ap-northeast-2`)
- **외부 API**: Google Maps Places, Kakao Mobility, TourAPI (전부 백엔드에서만 호출, 키는 환경변수)
- **실시간**: SSE (그룹 동기화)

### 중요한 특징 (배포 시 영향 주는 것들)
- **앱(백엔드/프론트)용 Dockerfile이나 GitHub Actions(CI)는 아직 없음.** docker-compose에는 Postgres만 있음.
  → 그래서 가장 단순한 방식인 **"EC2에서 jar 직접 실행 + nginx로 프론트 서빙/리버스 프록시"**로 배포하려고 함.
- 환경변수는 `backend/.env`, `frontend/.env` 파일로 주입 (Spring은 `spring.config.import`로 `.env`를 읽음).
- **SSE 때문에 nginx에서 `proxy_buffering off` 필요** (안 하면 실시간 동기화 멈춤).
- **refresh 토큰이 쿠키(Secure)라서 운영은 HTTPS여야 로그인이 유지됨.**
- OAuth redirect URI를 Google/Kakao 콘솔에 **운영 도메인 기준**으로 등록해야 함.

## 2. 배포 전체 계획 (5단계)

| 단계 | 할 일 |
|---|---|
| 1 | EC2(서버 컴퓨터) 켜기 + SSH로 접속 |
| 2 | EC2 안에 도구 설치 (JDK 21, nginx, docker) |
| 3 | ③ DB(PostgreSQL) docker-compose로 켜기 |
| 4 | ② 백엔드 빌드 + 실행 (`backend/.env` 운영값 작성, fat jar 실행) |
| 5 | ① 프론트 빌드 → nginx로 서빙 + 리버스 프록시 + HTTPS(certbot) |

## 3. 지금까지 진행한 상황 (★ 여기서 이어서 도와줘)

- AWS 계정: **새로 만들었음 (완전 백지 상태)**
- ④ S3: **이미 만들어둠** (리전 ap-northeast-2)
- 지금 막 **1단계(EC2 켜기)**를 하려는 참이야.

### 방금 안내받은 1단계 내용 (이대로 하는 중)
1. 리전을 **아시아 태평양(서울) ap-northeast-2**로 맞추기
2. EC2 → "인스턴스 시작"
3. 설정:
   - 이름: `enjoy-trip`
   - OS: **Ubuntu Server 24.04 LTS** (프리 티어)
   - 인스턴스 유형: `t2.micro` 또는 `t3.micro` (프리 티어)
   - 키 페어: 새로 생성 (`enjoy-trip-key`, RSA, .pem) → **.pem 파일 다운로드 후 잘 보관**
   - 네트워크: SSH / HTTP / HTTPS 트래픽 허용 체크
4. "인스턴스 시작" → "실행 중(Running)" 되면 1단계 완료

### 다음에 할 것
- 인스턴스가 "실행 중"이 되면, **그 다음은 EC2에 SSH로 접속(.pem 파일로 까만 터미널 들어가기)**이야.
- 참고로 내 PC는 **Windows**야 (SSH 접속 방법 알려줄 때 참고해줘).

## 4. 너에게 부탁

위 1단계가 끝났다는 가정 하에, **"EC2에 SSH로 접속하기"부터 한 단계씩** 이어서 알려줘.
- Windows 기준으로 알려줘 (PowerShell의 ssh 명령 또는 다른 방법).
- .pem 파일 권한 문제 같은 초보자가 흔히 막히는 부분도 미리 짚어줘.
- 한 번에 한 단계씩! 내가 "됐어" 하면 다음으로.
