## 1. DB Schema.sql (테이블 생성 · 초기 데이터)

| 항목 | 위치 |
| --- | --- |
| 통합 스키마(최종 상태, 제출용) | [`schema.sql`](schema.sql) |
| 운영 원천(버전관리) | `backend/src/main/resources/db/migration/V2 ~ V29` (Flyway) |

- 운영 환경은 **Flyway**가 기동 시 `V2~V29`를 순차 적용한다. (`flyway.enabled=true`)
- 제출용 `schema.sql`은 모든 마이그레이션을 적용한 **최종 형태 + 초기 데이터(설문 12문항)**를 한 파일로 정리한 것이다.
- 초기 데이터는 애플리케이션 기동 시 `DataInitializer`가 멱등 시드(설문 문항 12개 + 테스트 계정 `test@test.com / test1234`).

```bash
# 수동 적용 예시 (psql)
psql -U enjoy_trip_user -d enjoy_trip -f docs/제출물/소스코드/schema.sql
```

## 2. Spring Boot 프로젝트 소스 — `backend/`

```
backend/
├── build.gradle                 # Spring Boot 4.0.6 / JDK21 / QueryDSL / JWT / AWS S3
├── Dockerfile
└── src/main/
    ├── java/com/enjoytrip/backend/
    │   ├── domain/              # 도메인별 패키지 (controller/service/repository/entity/dto)
    │   │   ├── auth/  survey/  group/  place/  schedule/  vote/
    │   │   ├── expense/  settlement/  accommodation/  recommend/
    │   │   ├── notification/  sse/  gallery/  home/  mypage/
    │   └── global/             # 공통
    │       ├── config/         # Security, CORS, S3, OpenAPI, DataInitializer
    │       ├── security/       # JWT 필터/유틸, OAuth2 핸들러
    │       ├── crypto/         # PayoutCipher (AES-GCM 컬럼 암호화)
    │       ├── event/          # DomainEvent, EventType (SSE 계약)
    │       ├── exception/      # BusinessException, ErrorCode, GlobalExceptionHandler
    │       ├── response/       # ApiResponse
    │       └── storage/        # ObjectStorageService (S3)
    └── resources/
        ├── application.yml          # 공통 설정
        ├── application-prod.yml     # 운영(HTTPS) 프로필
        └── db/migration/           # Flyway V2~V29
```

- **레이어드 아키텍처**: Controller → Service(@Transactional) → Repository(JPA+QueryDSL) → Entity.
- **권한**: `@RequiredGroupMember` / `@RequiredGroupOwner` AOP + 서비스 레벨 검증.
- **외부 연동**: Google Places / Kakao Mobility / TourAPI / Gemini 클라이언트는 BE 프록시 + DB 캐시.

### 빌드/실행
```bash
cd backend && ./gradlew bootJar          # 빌드
docker compose -f docker-compose.backend.yml up -d --build   # 컨테이너 실행(:8080)
# API 문서: http://localhost:8080/swagger-ui/index.html (운영 프로필에선 비활성)
```

## 3. 프론트엔드 프로젝트 소스 — `frontend/` (과제표의 "Vue.js" 칸 = React)

```
frontend/
├── package.json            # React 19 / TS 6 / Vite 8 / Tailwind 4 / TanStack Query / Zustand
└── src/
    ├── App.tsx             # 라우팅 (react-router-dom 7)
    ├── api/                # axios 클라이언트(도메인별), 인터셉터 토큰 재발급
    ├── pages/              # 화면 (auth/home/survey/group/place/schedule/vote/expense/...)
    ├── components/         # 공통 컴포넌트 (Button/Modal/Toast/Skeleton/EmptyState/AppLayout)
    ├── store/              # Zustand (auth 메모리 토큰, settings)
    ├── lib/                # kakaoMap, naver, cn 등 유틸
    └── types/              # 도메인 타입
```

### 빌드/실행
```bash
cd frontend
npm install
npm run dev        # 로컬 개발 (Vite)
npm run build      # 프로덕션 빌드 (Vercel 배포)
```

> ⚠️ 과제 예시 스택은 Vue.js지만, **팀 추가 아이디어 구현** 규정에 따라 **React 19 + TypeScript**로 구현했다.

## 4. 활용 데이터셋

본 프로젝트는 정적 데이터셋을 적재하기보다 **외부 OpenAPI를 실시간 호출 + DB 캐시**하는 방식이다.

| 데이터셋 / API | 용도 | 비고 |
| --- | --- | --- |
| **Google Maps Places** | 모든 장소(숙소·맛집·카페·명소·쇼핑) 검색·상세·사진 | 단일 소스, 24h/7d 캐시 |
| **카카오 모빌리티** | 자동차/대중교통/도보 이동 시간·비용(톨비·연료비·택시비) | 1h 캐시 |
| **TourAPI (한국관광공사)** | 지역·성향 기반 관광지 추천 | 24h 캐시, 공공데이터포털 |
| **Gemini (Google AI)** | 장소 리뷰 AI 요약 | 선택 기능 |
| **시·도/시·군·구 행정구역** | 목적지 선택 드릴다운 | 프론트 정적 데이터 |
| **설문 문항 마스터(12)** | 성향 5차원 환산 | `schema.sql` seed / `DataInitializer` |
| **그룹 커버 프리셋(8)** | 그룹 커버 그라데이션 | 프론트 정적 |

> 모든 외부 API 키는 백엔드 `.env`(gitignore)로만 주입하며 커밋·프론트 노출 금지.
