# 요구사항 정의서

**프로젝트명** 그룹 여행 협업 플랫폼 (enjoy-trip)
**문서버전** v1.1 (구현 반영본) · 2026-06-25
**원천 문서** `docs/requirements.md` (SRS v1.1)

---

## 1. 개요

본 시스템은 친구·가족 등 2~8인이 **하나의 여행을 함께 계획·결정·정산**하는 협업 플랫폼이다.
Wanderlog를 벤치마크하되, 국내 사용자를 위해 다음 5가지를 차별점으로 구현했다.

| # | 차별점 | 구현 요구사항 |
| --- | --- | --- |
| 1 | 그룹 의사결정 강화 | FR-VOTE-01~04 (장소 후보 투표·실명·자동채택) |
| 2 | 성향 기반 매칭 | FR-SURVEY-01~03 (5차원 벡터·코사인 유사도 페르소나) |
| 3 | 국내 최적화 장소/이동 | FR-PLACE(Google Places 단일소스) + FR-SCHEDULE-04(카카오 모빌리티 비용) |
| 4 | 한국형 정산 | FR-EXPENSE-04~07 (최소송금 Greedy·토스/카카오페이 딥링크) |
| 5 | 실시간 동기화 | FR-SSE-01~04 (SSE 이벤트·React Query 캐시 무효화·폴백) |

요구사항 ID 규칙: `FR-{도메인}-{번호}`(기능), `NFR-{카테고리}`(비기능), `EI-{번호}`(외부 인터페이스).

## 2. 외부 인터페이스 요구사항 (EI)

| ID | 소스 | 담당 영역 | 핵심 규칙 |
| --- | --- | --- | --- |
| EI-01 | **Google Maps Places** | 모든 장소 검색(숙소·맛집·카페·명소·쇼핑) | 카테고리는 `includedType`으로 필터, 필드마스크 명시, 검색 24h·상세 7d DB 캐시 |
| EI-02 | **카카오 모빌리티** | 이동 시간·비용(자동차/대중교통/도보) | 톨비·연료비(거리/13km/L×1700원)·택시비·운임, 경로 1h 캐시 |
| EI-03 | **TourAPI(한국관광공사)** | 성향 기반 관광지 추천 | 지역기반 목록, 24h 캐시 |
| EI-04 | **토스/카카오페이** | 송금 딥링크/QR | URL Scheme 생성만(`supertoss://`, `kakaopay://`), 실제 송금은 사용자 앱 |
| EI-05 | **Google/Kakao OAuth2** | 소셜 로그인 | Authorization Code, 백엔드 OAuth2 Client, 1회용 코드로 프론트 전달 |
| EI-06 | **Gemini (Google AI)** | 장소 리뷰 AI 요약 | 미설정 시 해당 기능만 비활성 |

> 보안 원칙: 모든 외부 API 키는 **백엔드 환경 변수**로만 사용하고 프론트에 노출하지 않는다.

## 4. 비기능 요구사항 (NFR)

| 카테고리 | 요구사항 | 구현 |
| --- | --- | --- |
| NFR-PERF | API p95 500ms, SSE 1s, 1그룹 8명 | 외부 API DB 캐시(검색24h/상세7d/경로1h), N+1 제거, 인덱스 |
| NFR-SEC | HTTPS, JWT 32B+, SQLi/XSS 방지, 키 비노출 | Nginx+Let's Encrypt, JPA 바인딩, React 이스케이프, AES-GCM 계좌 암호화 |

| NFR-REL | 의미있는 에러, 트랜잭션 단위, 정산 멱등 | `GlobalExceptionHandler`+`ApiResponse`, `@Transactional`, 멱등 등록 |
| NFR-MAINT | 도메인 패키지, Flyway, Conventional Commits | `domain/*` 구조, V2~V29 마이그레이션 |

## 6. 상태 전이

- **그룹**: `PLANNING → IN_PROGRESS → COMPLETED → (DELETED)`
- **일정**: `PLANNED ↔ VOTING → (CANCELLED)`
- **정산 송금**: `PENDING → SENT → COMPLETED`
- **숙소**: `SELECTED → BOOKED`
