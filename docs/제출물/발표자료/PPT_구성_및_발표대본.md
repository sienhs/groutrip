## 슬라이드 8. 화면 흐름도 및 시연 (핵심!)

화면 흐름도(= `06_화면설계서.md §2`) + 아래 **시연 시나리오 6단계**.

### 시연 시나리오 (라이브 데모 대본)

1. **로그인** — 카카오 소셜 로그인 → 홈 진입
2. **그룹 생성 + 초대** — "제주도 여행" 생성, 초대코드 복사 → (2번째 계정) 참여 → *실시간 멤버 합류 토스트*
3. **장소 검색·보관함** — "제주 흑돼지" 검색(Google) → 보관함 추가 → 리뷰 AI 요약 보기
4. **일정 배치 + 이동/지도** — 일정 3개 배치, 드래그로 순서 변경, 이동 3탭(자동차 비용), 일정 지도 확인
5. **투표** — 한 슬롯을 빈 일정으로 두고 후보 2개 등록 → 양쪽 계정 투표 → 마감·자동 채택 (*실시간 반영*)
6. **정산** — 지출 등록(균등/비율) + 이동·숙박비 자동등록 → 정산 매트릭스(최소 송금) → 토스 딥링크 → 완료 처리

> 시연 팁: 두 브라우저(또는 시크릿창)를 나란히 띄워 **SSE 실시간 반영**을 보여주면 임팩트가 크다.

## 슬라이드 9. 적용 패턴 및 핵심 알고리즘

- **레이어드 + 도메인 패키지** 아키텍처, **AOP 권한 검증**(@RequiredGroupMember/Owner)
- **이벤트 기반 SSE**: Service가 `ApplicationEvent` 발행 → `SseEventBridge`가 브로드캐스트(느슨한 결합)
- **외부 API DB 캐시 전략**: 검색24h/상세7d/경로1h/추천24h (비용·지연 동시 절감)
- **핵심 알고리즘 ①: 최소 송금 횟수 (Greedy)**
  1. 멤버 잔액 = 낸 돈 − 부담액
  2. 채권(+)/채무(−) 분리, 절댓값 내림차순
  3. max+ ↔ max− 매칭, min(절댓값)만큼 송금 기록
  4. 잔액 0 제외 반복 → 송금 횟수 최소화
- **핵심 알고리즘 ②: 성향 코사인 유사도** — 5차원 벡터 평균 + 멤버 간 코사인으로 그룹 일치율·충돌 차원 산출
- **보안**: AES-GCM 컬럼 암호화(정산 계좌), JWT, OAuth2, HTTPS

> ✅ 클래스/시퀀스 다이어그램 = `03_클래스다이어그램.md`, `07_…§4` 이미지 삽입.

# 부록 A. AI 사용 보고서

> 제출형식: 발표 PPT에 작성 가능 (또는 별도 첨부). 아래를 슬라이드/표로 옮긴다.

## A-1. 사용 도구
- **Claude Code (Anthropic)** — 코드 작성·리팩터링·문서화·디버깅 페어프로그래밍
- (보조) GitHub Copilot / ChatGPT 등 사용 시 함께 기재

## A-2. 활용 영역
| 영역 | 활용 내용 |
| --- | --- |
| 설계 | 요구사항 정리, ERD/클래스 다이어그램 초안, 외부 API 책임 분리 전략 |
| 구현 | 도메인 CRUD, QueryDSL N+1 제거, SSE/정산 알고리즘, OAuth/JWT 배선 |
| 디버깅 | Flyway 오토컨피그, 마이그레이션 충돌, 쿠키/CORS/SSE 프록시 이슈 |
| 인프라 | Docker/Nginx/HTTPS 배포 가이드, S3 전환 |
| 문서 | 산출물(요구사항/ERD/화면설계/발표자료) 작성 |

## A-3. 대표 프롬프트 예시
- "Spring Boot 4 + JPA로 그룹 비용 정산의 최소 송금 횟수 Greedy 알고리즘을 구현해줘. 잔액 계산과 분담 테이블 기준."
- "Vercel(HTTPS) 프론트와 EC2 백엔드를 Nginx 리버스 프록시 + Let's Encrypt로 HTTPS 전환하는 단계별 가이드와 설정 파일을 만들어줘."
- "일정 지도의 이동 경로 선이 직선이라 꺾임이 어색해. 일자별로 곡선으로 부드럽게 그려줘."

## A-4. 검증 원칙
- AI 산출물은 **컴파일·테스트·실제 시연으로 검증** 후 반영.
- 비밀값(API 키·시크릿)은 AI에 노출하지 않고 환경 변수로만 관리.
- 외부 API 캐시·권한 검증 등 비즈니스 규칙은 사람이 최종 확인.
